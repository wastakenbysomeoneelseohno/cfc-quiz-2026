QL — Quiz Language

The complete spec for a QL deck — a plain .txt file. QL is line-oriented: the first character of each line decides what it is, so a parser reads it in one pass (see ql.js, ~95 lines, no dependencies). A worked deck is in sample-deck.txt.

Two ideas carry the whole language:

  1. The answer's shape decides the question type — you never name a type.
  2. Each separator is used only where it can't collide — options split on |, settings split on ,. Since real answers contain commas (love, joy, peace) but never pipes, this means content almost never needs escaping.

1. Lines at a glance

Line starts with Kind Section
# deck name §2
## stream name §2
### slide (### (type) … = special slide) §5 / §6
key: field — teams:, theme:, image: §2 / §6
> settings (comma-separated) §4
< the answer (pipe-separated) §6
--- / === comment / divider — any line starting with --- or === is ignored §8
\ escape — the rest of the line is literal body text (not a comment/field/sigil) §8 / §9
anything else body text (a question prompt, or a Title slide's body) §6

2. Structure & fields

# CFC Sample Deck
teams: 20
theme: midnight
header: CFC Quiz Night
footer: Calvary Fellowship, 2026
header-logo: data:image/svg+xml;base64,…
footer-logo: data:image/svg+xml;base64,…

## Main Quiz
Command Means Values Default
# [name] deck name (file name + browser tab) any text Untitled Deck
## [name] a stream (quiz track; same teams, shared scoreboard) any text —
teams: [n] team count, shared by all streams 1–25 20
theme: [name] theme (deck-level, or stream-level under a ##) a §7 name midnight
header: [text] header title shown on every slide any text hidden (no fallback)
footer: [text] footer note (bottom-right, every slide) any text hidden
header-logo: [data-url] header logo (embedded) a base64 data: URL hidden
footer-logo: [data-url] footer logo (embedded) a base64 data: URL hidden
sounds: [on/off] countdown sounds (3·2·1·ring) on every timed question on / off on (add sounds: off to silence)

Streams merge by name on import: same name replaces, new name adds.

2.1 Header & footer (branding)

header: + header-logo: show centered in a band at the top of every slide; footer: + footer-logo: sit at the bottom-right. The header is fully optional — nothing set means nothing is drawn (no deck-name fallback). Logos and all images are embedded base64 data: URLs, never external file references — a deck is one self-contained file that works fully offline. Keep each small (an SVG, or a PNG under ~50 KB — header logo ≤44 px tall, footer ≤26 px).

In the footer text you can place the footer logo inline with the token <IMG> (e.g. footer: Hosted by <IMG> CFC); without <IMG> the logo just goes at the end (the right side).

2.2 Team names — ## (teams)

By default the teams are just Team 1 … Team N (where N is teams:). To give them real names, add a ## (teams) block — one name per line, in team order. A blank line keeps that team's default (Team 3), so you can name only the teams you care about:

# CFC Quiz
teams: 6
## (teams)
The Eagles
Lions

Sharks

Here team 3 stays Team 3 and teams 5–6 stay Team 5 / Team 6 (only the named prefix is needed). Names are deck-level (shared by every stream, like teams:) and show everywhere — the score sheet, leaderboard, and podium. They travel in the JSON and .txt exports; the Display also lets you review/edit them on the start screen before the show. A deck with no ## (teams) block simply keeps Team N (fully backward compatible).

3. The answer line <

One line per question, starting with <. Options separate with |; the content decides the type.

You write Type
< 27 short answer
< love, joy, peace short answer — commas are free, no escaping
< Buzzer | Verses | Pictures poll (no *)
< Spirit | grace | Lord + an answer: line Options + Answer — options shown (auto-lettered A/B/C), free-text answer (§6.1)
< *Noah | Moses | David choices — one *
< *Matthew | *Mark | Acts pick-all — many *
< *True | False true / false (auto-detected)
< Moses -> Led the Exodus | David -> King of Israel matching

Markers: * (prefix) = the correct option · | separates options · -> = maps-to. A body line above the < turns a short answer into a titled question (§6).

Example

### Who built the ark?
< *Noah | Moses | David | Paul

4. Settings — the > line

One blockquote-style line per slide (or stream), > token, token, …. Split on commas, then match each token. On a stream it's the defaults for new questions; on a question it overrides them. Omitted tokens keep their default.

Token Regex Means Default
[N]s ^\d+s$ countdown of N seconds 20s
[M]:[SS] ^\d+:\d{2}$ countdown as m:ss (60s+) —
no-timer ^no-timer$ no countdown —
[N]pts ^\d+pts$ worth N points (≥ 0; ✓ adds, ✗ subtracts) 1pts
scored ^scored$ has a scoring page ✓
unscored ^unscored$ no scoring page —
reveal ^reveal$ show an answer screen —
no-reveal ^no-reveal$ no answer screen ✓
auto-transition ^auto-transition$ advance itself when the timer hits 0 —
manual-transition ^manual-transition$ wait for the operator to advance ✓
timer-auto ^timer-auto$ timer starts automatically ✓
timer-manual ^timer-manual$ timer waits for ▶ Start —
loop-enabled / loop-disabled ^loop-(enabled|disabled)$ (Reveal Answers) wrap after the last loop-disabled
additional / standard ^(additional|standard)$ a hidden reserve question, swapped in on a disqualify — see §10 standard

Baseline: 20s, 1pts, scored, no-reveal, manual-transition, timer-auto.

4.1 Timer values

no-timer, or: 10 15 20 25 30 35 40 45 50 55 60 (every 5s), 75 90 105 120, then 150 180 210 240 270 300 330 360 390 420 (every 30s). 60s+ may be written m:ss.

Example

> 25s, 5pts, scored, reveal

5. Special slides — ### (type) …

A ### whose text starts with a (type) tag is that slide. An unknown type is an error (the import stops with the line). Any ### without a tag is a question (§6).

Tag Slide Below it
### (title) [text] section / round divider sub: (optional subheading) + image: (shown above) + body lines = body
### (leaderboard) [text] current standings —
### (podium) [text] top-3 finale —
### (reveal-answers) [text] answer browser a > line (§5.1)
### (note) [text] standalone reveal slide a body line = body
### (buzzer) [text] first-to-buzz question a < answer

A title slide takes the ### (title) line as the heading, an optional sub: line as the subheading (a short second line), an optional image: (shown above the heading), and any remaining lines as the body (longer text — kept across line breaks):

### (title) Round 2 — Verses
sub: Scripture round
image: data:image/svg+xml;base64,…
Read each verse aloud
before the team answers

5.1 Reveal Answers settings (its > line)

Token Means Default
[N]s seconds per answer (0s = manual) 10s
loop-enabled / loop-disabled wrap after the last answer loop-disabled
additional also browse the reserve (additional) questions' answers off — standard questions only

Example

### (reveal-answers) The Answers
> 8s, loop-enabled

6. Questions

A plain ### is a question; its type comes from the < answer line (§3). Optional parts:

  • a body line between the heading and the < → makes it a titled question;
  • an image: [data-url] field → adds an embedded picture (with a typed answer it's an image question; with options it's a multiple choice that shows a picture);
  • a > settings line → per-question overrides.

Examples

### How many books are in the New Testament?
< 27

### John 3:16
For God so loved the ___.
< world

### Name this symbol.
image: data:image/svg+xml;base64,…    (embedded — no external files)
< The cross

### Quick warm-up: name a disciple.
< Peter
> 10s, unscored, no-reveal

6.1 Options + Answer (a list with a free-text answer)

A < options list with no * plus an answer: line is an Options + Answer question. The options show as auto-lettered tiles (A, B, C…) like multiple choice, but there's no correct option to pick — instead the free-text answer on the answer: line is revealed (replacing the options) when you advance. Perfect for "arrange in order" or "identify each one".

### Arrange the "one" attributes of Ephesians 4:4-6 in order.
< Spirit | grace | Lord | body | God and Father | hope | baptism | faith | peace
answer: D, A, F, C, H, G, E

A list with no * and no answer: stays a poll (unchanged) — adding the answer: line is what makes it Options + Answer.

7. Themes

Reference a theme by name with theme: on the deck (#) or a stream (##).

7.1 Built-in themes

midnight (default) · royal · parchment · sunrise.

## Audience Round
theme: parchment

7.2 Custom theme block — ## (theme) [name]

Define a theme once, then reference it by name anywhere. Each field is a key: value line.

Field Means Values
bg: [type], [c1], [c2], [angle] background gradient radial/linear, two hex colors, an angle
ink: [hex] text color hex
accent: [hex] accent color hex
correct: [hex] correct-answer color hex
heading-font: [name] heading typeface System Sans, Georgia Serif, Helvetica, Trebuchet, Palatino Serif, Verdana
body-font: [name] body typeface (same list)

8. Escaping & rules

  • Comments / dividers: any line that starts with --- or === is ignored, so you can break a deck into sections — === Round 2 ===, --- audience questions ---, or a bare ------ rule. (Only the start of the line matters, so --- or === mid-text in a question or answer is kept as normal content.)
  • Escape inside an answer with \: \| (literal pipe), \* (literal leading star), \-> (literal arrow — not a matching pair), plus \, \< \\. A trailing * is already literal — only a leading * marks the correct option.
  • **Escape a whole line with a leading \:** any line that would otherwise be read as a comment, a field, or a sigil becomes **literal body text** — \--- not a divider, \theme: my text, \> a quote, \< text, \# hashtag. (Write \\… for a body line that really begins with a backslash.)
  • Type is inferred, never declared — the < line's shape decides a question's type.
  • One line = one thing. The parser classifies by the first character, so every error carries its line number.
  • A malformed file imports nothing and reports one located error (file + line).
  • QL carries questions, answers, settings, and theme names — not custom themes or password encryption; use the 🔒 JSON save for those.

9. Gotchas — and how to escape them

QL reads a question's type from the shape of its answer — that's why you never label types, but it also means a stray symbol can change the type. Every case below is escapable:

If you write… QL reads it as… Get what you meant with…
< Moses -> the people a matching question < Moses \-> the people
< rock | roll (one answer with a pipe) a 2-option poll put a \ before the pipe
< *5 stars (literal leading star) first option marked correct < \*5 stars
a body line starting theme: teams: sounds: sub: answer: image: header: footer: a field (body text lost) start it with \ → \theme: …
a body line starting > < # --- === settings / answer / heading / comment start it with \ → \> a quote

Not footguns — by design, and never silent-wrong:

  • A list with no * is a poll. That's the definition of a poll (no right answer). If you meant a scored question, mark the correct option with *.
  • Two options reading True / False become a true/false question — and a true/false with no * is a located error (you must mark the answer), never a silent false.
  • A question with both an image and a body line is a located error — no slide type carries both, so it's rejected rather than quietly dropping the body.

10. Additional questions (additional) & disqualifying

Mark a question additional (a > additional token, or the Maker's Additional question → Yes toggle) to make it a hidden reserve. Additional questions keep their slide number in the editor but are skipped in the show by default.

During the show — in both the Maker preview and the Display — the operator's 🚫 button disqualifies the question currently on screen; pressing it again re-enables that question (it works like a checkbox). A disqualified question stays on screen with a big ✗. The rule:

  • Disqualifying a question pulls it out of the count and brings in the next hidden additional question — so the number in play stays the same (one out, one off the bench).
  • Re-enabling reverses it exactly: the question returns and the last extra question re-hides.

Example — Q1 Q2 Q3 plus additional A1 A2:

default              → Q1 Q2 Q3
disqualify Q2        → Q1 Q3 A1
disqualify Q2 + Q3   → Q1 A1 A2
un-disqualify Q3     → Q1 Q3 A1   (A2 goes back to the bench)

additional is authored (it travels in the .txt and the JSON save). Disqualifying is a runtime action — it is not a QL token; it persists in the 🔒 JSON save and across a Display reload (resume), but is never written into a .txt.
